This guide will go over how to install Valkey GLIDE for your specific system.

## System Requirements

The release of Valkey GLIDE was tested on the following platforms:

### Linux

* Ubuntu 20 (x86\_64/amd64 and arm64/aarch64)
* Amazon Linux 2 (AL2) and 2023 (AL2023) (x86\_64)

<details>
  <summary>`glibc` requirements</summary>
  Node.js requires `glibc` 2.17+. This should be the standard for GNU Linux based systems.

  However, to check you can use:

  ```
  ldd --version
  ```
</details>

:::caution
Currently Alpine Linux / MUSL is NOT supported.
:::

### MacOS

* macOS 14.7 (Apple silicon/aarch\_64)
* macOS 13.7 (x86\_64/amd64)

### Windows

Windows support is currently available only for Valkey-GLIDE Java

* Windows (x86\_64)

## Installing and Setting Up

<Tabs syncKey="progLangInExamples">
  <TabItem label="Python">
    **Requirements:** Python 3.9 to 3.13.

    To get started, GLIDE Python is available on [pypi]().

    ```bash
    pip3 install valkey-glide
    ```

    For the synchronous Python client, use

    ```bash
    pip3 install valkey-glide-sync
    ```

    :::note
    Valkey GLIDE transparently supports both the `asyncio` and `trio` concurrency frameworks.
    :::
  </TabItem>

  <TabItem label="Java">
    **Requirements:** JDK 11+

    To get started, add one of the following snippet into the build file for your tool.

    :::caution[Include Classifiers]
    You must include the classifier for your OS. It is recommended to use the [OS Detector](https://github.com/google/osdetector-gradle-plugin) plugin to
    detect the correct classifier for your OS.

    Otherwise, the following specifiers are supported:

    ```
    osx-x86_64
    linux-aarch_64
    linux-x86_64
    linux_musl-aarch_64
    linux_musl-x86_64
    windows-x86_64
    ```

    :::

  <TabItem label="Node">
    **Requirements:** Node.js 16+

    **For npm users on Linux it is recommended to use npm >=11**. It supports optional downloads based on libc. Yarn users should not be concerned.

    :::note
    The library is dependent on the [protobufjs library](https://protobufjs.github.io/protobuf.js/#installation), which adds size to the package. The package is using the protobufjs/minimal version, hence, if size matters, bundlers should be able to strip the unused code. It should reduce the size of the dependency from 19kb gzipped to 6.5kb gzipped.
    :::

    To get started, Valkey GLIDE Node.js is available on npm for install.

    ```bash
    npm install @valkey/valkey-glide
    ```
  </TabItem>

  <TabItem label="Go">
    **Requirements:** Go 1.22+

    ```bash
    # Install GLIDE
    go get github.com/valkey-io/valkey-glide/go/v2

    ```
  </TabItem>

  <TabItem label="PHP">
    **Requirements:** PHP 8.1+

    Install using pie (recommended):

    ```bash
    pie install valkey-io/valkey-glide-php
    ```

    Or install from PECL:

    ```bash
    curl -L https://github.com/valkey-io/valkey-glide-php/releases/download/v1.0.0/valkey_glide-1.0.0.tgz -o valkey_glide-1.0.0.tgz
    pecl install valkey_glide-1.0.0.tgz
    ```

    Enable the extension in your `php.ini`:

    ```ini
    extension=valkey_glide
    ```

    Verify installation:

    ```bash
    php -m | grep valkey_glide
    ```
  </TabItem>
