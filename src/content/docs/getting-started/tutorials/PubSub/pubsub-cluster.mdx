This tutorial guides you through building a simple real-time messaging app with Valkey GLIDE Python. 

By the end, you will learn:
  * How to set up a multi-node Valkey cluster in Docker for testing.
  * Creating a simple messaging app example using Valkey pubsub.

### The Feature: Resilient PubSub in Valkey GLIDE

When a primary node fails in a cluster, a replica node will take its place changing the **topology**.
GLIDE's PubSub model is designed for this this in mind.

  * **Automatic Resubscription:** GLIDE is responsible for tracking topology changes in real time. If the node you're subscribed to fails, GLIDE automatically detects the new primary and resubscribes to the correct node for you.
  * **Unified Interface:** The client provides a coherent interface, minimizing the differences between connecting to a single node or a massive cluster.
  * **Immutable Configuration:** You define your subscriptions when you create the client. This immutable state allows GLIDE to know exactly what to resubscribe to after any disconnection or topology change.

:::tip
It is a best practice to use a **dedicated client instance for your PubSub subscriptions**. 
Your application will have one client for listening (the subscriber) and another for all other commands, including publishing.
:::

## Prerequisites

Before you begin, you'll need a few things:

1.  **Python:** Valkey supports Python 3.9 to 3.13.
2.  **Docker:** Have docker installed.

## Setting up Valkey Cluster

First, we'll create a 3-node Valkey cluster in Docker.
The `devcontainer` is used to run the Valkey GLIDE clients. If we run the clients from the host machine, it
will not be able to resolve the network addresses of the cluster which will result in timeout errors.

1.  Create `docker-compose.yml`
    <details>
    <summary>`docker-compose.yml`</summary>
      ```yaml
      services:
        # Container for running client scripts.
        devcontainer:
          image: python:3.11-slim
          container_name: devcontainer
          working_dir: /tutorial
          volumes:
            - .:/tutorial
          networks:
            - valkey-net
          command: sh -c "pip install valkey-glide && tail -f /dev/null"

        valkey-1:
          image: valkey/valkey
          container_name: valkey-1
          command: >
            valkey-server --port 6379 --cluster-enabled yes --cluster-config-file /data/nodes.conf
            --cluster-node-timeout 5000 --cluster-port 16379 --appendonly yes --bind 0.0.0.0
          ports:
            - "7001:6379" # Kept for optional host access
          volumes:
            - valkey-data-1:/data
          networks:
            - valkey-net

        valkey-2:
          image: valkey/valkey
          container_name: valkey-2
          command: >
            valkey-server --port 6379 --cluster-enabled yes --cluster-config-file /data/nodes.conf
            --cluster-node-timeout 5000 --cluster-port 16379 --appendonly yes --bind 0.0.0.0
          ports:
            - "7002:6379"
          volumes:
            - valkey-data-2:/data
          networks:
            - valkey-net

        valkey-3:
          image: valkey/valkey
          container_name: valkey-3
          command: >
            valkey-server --port 6379 --cluster-enabled yes --cluster-config-file /data/nodes.conf
            --cluster-node-timeout 5000 --cluster-port 16379 --appendonly yes --bind 0.0.0.0
          ports:
            - "7003:6379"
          volumes:
            - valkey-data-3:/data
          networks:
            - valkey-net

      volumes:
        valkey-data-1:
        valkey-data-2:
        valkey-data-3:

      networks:
        valkey-net:
          driver: bridge
      ```
    </details>
2.  Start the nodes with `docker-compose`

    ```bash
    docker-compose up -d
    ```

3.  Finally initialize the cluster

    ```bash
    docker exec valkey-1 valkey-cli --cluster create \
      valkey-1:6379 valkey-2:6379 valkey-3:6379 \
      --cluster-yes
    ```

## The Messaging App

The example will have two scripts, a `listener.py` and `producer.py`. 
Place them in the same folder as the `docker-copmose.yml`.

### The Listener
`listener.py` will listen for messages from the Valkey cluster using Pub/Sub until interupted.

It connects to the cluster and subscribes to the messaging channel. 
It uses a callback function to handle the message, printing the message's content.

<details>
<summary> `listener.py` </summary>
```python
import asyncio
import typing
from glide import (
    GlideClusterClient,
    NodeAddress,
    GlideClusterClientConfiguration,
)
from glide.async_commands.core import PubSubMsg

# This callback function will be triggered for every message
def callback(msg: PubSubMsg, context: typing.Any):
    """
    Prints the received message and the context.
    The context is an optional object provided during configuration.
    """
    print(f"Callback context: {context}")
    print(f"Received message: '{msg.message}' from channel: '{msg.channel}'")

async def main():
    # Define the addresses of our cluster nodes
    # We only need to provide one or more nodes;
    # the client will discover the rest of the cluster topology.
    nodes = [
        NodeAddress(host="valkey-1", port=6379)
    ]

    # Set up the subscription configuration
    # We will listen to the 'notifications' channel
    subscriptions = GlideClusterClientConfiguration.PubSubSubscriptions(
        channels_and_patterns={
            # We use 'Exact' to subscribe to a specific channel name
            GlideClusterClientConfiguration.PubSubChannelModes.Exact: {"messages"}
        },
        callback=callback,
        context=None
    )

    # Configure the cluster client
    config = GlideClusterClientConfiguration(
        addresses=nodes,
        pubsub_subscriptions=subscriptions
    )

    client = None
    stop_event = asyncio.Event()

    try:
        print("Starting listener client...")
        # Create and connect the client
        client = await GlideClusterClient.create(config)
        print("Listener connected. Waiting for messages...")
        print("Press CTRL+C to stop.")

        # Wait indefinitely until an interrupt (like CTRL+C) is received
        await stop_event.wait()

    except KeyboardInterrupt:
        print("\nStopping listener...")
    except Exception as e:
        print(f"An error occurred: {e}")
    finally:
        if client:
            # Cleanly close the client.
            # This handles unsubscribing from the server.
            await client.close()
            print("Listener client closed.")

if __name__ == "__main__":
    asyncio.run(main())
```
</details>

### The Producer

Next, let's add the `producer.py`. It will send user inputs to the channel on the Valkey cluster.

<details>
<summary> `producer.py` </summary>
```python
import asyncio
import asyncio
from glide import GlideClusterClientConfiguration, NodeAddress, GlideClusterClient

async def main():
    # Cluster node addresses
    nodes = [
        NodeAddress(host="valkey-1", port=6379)
    ]

    # This client has NO subscription config
    config = GlideClusterClientConfiguration(addresses=nodes)

    client = None

    try:
        # Create and connect the client
        client = await GlideClusterClient.create(config)
        print("Producer connected.")

        # Publish an initial "Hello World" message
        channel = "messages"
        message = "Hello World!"
        print(f"Publishing: '{message}' to channel '{channel}'")
        await client.publish(message, channel)

        # The 'publish' command routes to the correct node automatically
        print("Enter messages to publish. Type 'exit' to quit.")

        # Loop and wait for user input
        while True:
            message = input("Message: > ")
            if message.lower() == 'exit':
                break

            # Publish the user's message
            await client.publish(message, channel)
            print(f"Published: '{message}'")

    except Exception as e:
        print(f"An error occurred: {e}")
    finally:
        if client:
            await client.close()
            print("Producer client closed.")

if __name__ == "__main__":
    asyncio.run(main())
```
</details>

### Running the Example

First, start `listener.py` on the `devcontainer`

```bash
docker compose exec devcontainer python listener.py
```

On a separate terminal, start `producer.py`

```bash
docker compose exec devcontainer python listener.py
```

Once started, you should see that the listener have recieved a message "Hello World!"

```bash
Starting listener client...
Listener connected. Waiting for messages...
Press CTRL+C to stop.
Callback context: None
Received message: 'b'Hello World!'' from channel: 'b'notifications''
```

Now messages you sent from the producer 

```bash
Message: > Valkey Glide Python
Published: 'Valkey Glide Python'
```

Will be received on the listener

```
Callback context: None
Received message: 'b'Valkey Glide Python'' from channel: 'b'notifications''
```
